## 一、这个产品是什么，为什么要做它

qtrade 是一套端到端的个人量化交易系统：从数据下载、信号计算、模拟盘执行、风险控制到资金分配，全部自动化，全部可审计。它管理的是你自己的钱，对标的是大私募和量化机构的**方法论**——不是它们的规模，而是它们赖以生存的那套东西：

> **规则先于结果，验证先于资金，纪律先于聪明。**

做它的根本原因是一个统计事实：散户亏钱的两大来源是**情绪**（追涨杀跌、砍赢单扛亏单）和**成本**（过度交易把手续费送给交易所）。机构的优势从来不是预测更准，而是系统性地消灭这两个漏洞。这套系统就是把"消灭漏洞的机制"写成代码。

## 二、系统的核心逻辑（三条设计信念）

### 信念 1：预注册纪律——先定及格线，再看考卷

系统里每一个策略上线前，都必须先把"什么算通过"写进 `research/log.md` 并冻结（例如"Sharpe ≥ 0.4 才立项"），然后才允许跑回测。**看到结果之后再定标准 = 自欺**，这是量化研究里最大的死因（行话叫过拟合/数据窥探）。两天前国内商品 CTA 终审 Sharpe 0.48 过线立项、昨天品种池扩展实验新品种无信号被拒——两个判决用的都是事前冻结的门槛，无论结果多诱人或多可惜，判决不含人的情绪。

54 个编号实验里，**被拒绝的远多于被采纳的**（立项/可用的只有个位数）。这不是失败率高，这是免疫系统在工作：每一次拒绝都写进日志连同"重开条件"，防止未来的自己在同一个坑里再掉一次。

### 信念 2：数据质量是第一 alpha

整个项目最大的单笔"收益"不来自任何聪明策略，来自两次数据修正：

- 美国期货：免费拼接数据 Sharpe 0.11 → 干净数据 0.31（E41）
- 国内商品：免费拼接数据 Sharpe 0.08 → 自建逐合约后复权 **0.48**（E50b）

同一个策略、同一批品种、同一套成本，只是把被"换月假跳空"污染的价格换成干净价格，结论就从"死刑"变成"立项"。为此系统自建了 2643 个真实合约文件的数据仓库和一套冻结规则的拼接引擎（主力合约=昨日持仓量最大、只向后换月、乘法后复权），研究和实盘执行共用同一份代码——**研究里验证的就是实盘里跑的，一个字都不差**（`parity` 命令可随时证明这一点，当前双账本漂移 0.0000）。

### 信念 3：成本结构决定构造

散户费率（万分之五到千分之一/边）下，任何"更平滑、更频繁调仓"的改进都会被手续费吃掉——这是四个被拒实验（参数集成/组合级波动率目标/基差套利/换手缓冲）用真金白银的回测反复证明的元结论。所以系统的形态是**粗参数 + 高调仓阈值 + 低换手**：crypto 账本典型持仓 9.3 天，国内商品日线级别，没有日内交易。这不是保守，是这个成本结构下的数学最优。

## 三、系统架构：五层，每层都有可验证的落地物

```
 信号层  →  执行层  →  风控层  →  组合层  →  验证层
```

**① 信号层（赚钱的来源）**
- crypto：趋势跟踪（三时间尺度投票）+ 均值回归（带牛熊闸门的超买超卖反转），各管一半风险，10 个主流币
- 国内商品：纯趋势（1/3/12 个月投票 + 波动率缩放），14 个品种跨五大板块，多空双向
- 每个信号都能用 `explain` 命令展示完整决策链——任何一笔持仓都答得出"为什么"

**② 执行层（少交学费）**
- 模拟盘引擎每小时自动运行（已装成 macOS 系统服务，关机重启都不影响）
- OKX 真实执行器已备好挂单优先模式（历史回放成交率 98%，成本从付费变返佣，省 ~0.6%/年），等 API key 即可启用
- 执行价格假设做过三角验证：收盘/开盘/结算价三种基准全部过立项线，不依赖任何单一假设

**③ 风控层（自动兜底，不需要你操作）**
- 数据陈旧 → 当天拒绝交易（宁可不动，不在坏数据上动）
- 单品种权重 ±25% 帽、总敞口 2 倍帽——正常运行永远碰不到，专门兜软件故障
- **回撤熔断**：任一账本回撤超过历史回测最大值的 1.5 倍（crypto 23%、商品 18.5%）→ 自动清仓并落下 HALTED 标记，人工审核前拒绝恢复交易。触发概率 0.4%/年——它存在的意义是"永不误触发、异常必拦截"

**④ 组合层（分散是唯一免费的午餐）**
- 两个进攻账本月收益相关性 **-0.10**：crypto 亏的年份商品往往在赚
- `allocate --capital 总额` 一条命令输出每账本该放多少钱（按实际波动率反比，每月重算）
- 组合层实测：可部署账本组合 Sharpe 0.96、最大回撤 -2.4%——高于任何单账本

**⑤ 验证层（防止系统腐烂）**
- 每周 `weekly`：账本体检 + 回测预期对照 + 制度到期提醒
- 每月 `revalidate`：策略是否退化的定量重检
- 每季度：品种池边际贡献重评 + 边缘方案复审
- 随时 `health`（数据完整性+心跳+熔断标记）和 `parity`（信号可复现性）

## 四、走过的路：54 个实验的编年史精华

**阶段一：主力策略的锻造与七年审判（E1-E29）**
从零构建 crypto 趋势+回归复合策略，然后做了整个项目最重要的一次验证：把历史拉回参数从未见过的 2019-2023 年（含 COVID 崩盘、双顶疯牛、大熊市），纯净样本外 Sharpe 1.11、最大回撤 10.5%。同期五种"机构式改进"全部未过预注册线被拒——策略原样保留，但证据等级从"调参样本"升级为"七年含纯净逆向检验"。

**阶段二：多市场远征与诚实判死（E19-E48）**
股票截面动量（三个宇宙）、A股价格因子、A股基本面因子、A股机器学习指增——逐一用数据判死或存档。特别值得说的是 A股 ML 线：LightGBM 挖出了真实的毛超额 +6.3%/年（两半段皆正），但散户成本吃掉一半，净收益不达标——**这条线的死因被精确量化为成本结构，而非信号不存在**，将来开通低佣通道可复活。为获得"此路不通"的机构级确定性花了 ¥200 买 Tushare 数据，这笔钱买断了几个月的自我怀疑。

**阶段三：国内商品 CTA 的死而复生（E50-E58）**
首测 Sharpe 0.08 差点判死，但按纪律怀疑数据而非策略 → 自建 2643 合约干净数据仓 → 终审 0.48 立项部署。随后一口气完成因子三件套判决（carry 已衰减、截面动量是趋势的劣化重复——只有趋势活着）、品种池扩展否决（广度的前提是新市场自带信号）、执行风险三查（价格基准/涨跌停锁板/换月成本全过）。

**阶段四：机构化基础设施（贯穿始终，本周末集中落地）**
风控闸门、体检命令、信号复现核验、资金分配命令、系统级心跳服务、风险画像（蒙特卡洛 5000 路径）、大白话决策手册——让系统从"一组策略"变成"一台可以交给时间运行的机器"。

## 六、诚实的预期管理（这部分最重要）

- **这不是印钞机。** 组合预期年化在中个位数（保守配置 ~3-6%），价值在于回撤浅（组合 maxDD 实测 -2.4%，单账本坏情况 -13%）和与大盘涨跌无关的绝对收益。
- **单账本必然有难看年份**：crypto 亏损年概率 28%，商品 37%。系统的答案不是预测哪年难看，而是让两本账的难看年尽量错开（相关性 -0.10）。
- **已知弱点已定价**：crypto 账本在单边暴涨年（如 2024）会跑输甚至小亏，约 5-7 年一遇；商品账本 2021 起的震荡市里基本白干。这些写在手册里，不是 bug，是这类策略的固有性格。
- **心理上最难的**：胜率只有 40%，靠尾部大赢单赚钱（前 10 笔大赢占净利 81%）。连亏是常态，系统靠的是不砍赢单、不错过行情、不手动干预。
